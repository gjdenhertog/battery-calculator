  <name>Task 1: Add same-origin SVG favicon and wire it into index.html</name>
  <files>public/favicon.svg, index.html</files>
  <action>
Create public/favicon.svg as a minimal, self-contained battery-glyph SVG suiting "Thuisbatterij Calculator". Hard constraints: a single inline `<svg>` with a viewBox and a small number of vector shapes (battery body rectangle, terminal/nub, and a simple charge indicator). NO external references of any kind — no `<image href>`, no remote fonts, no `xlink:href` to a URL, no `<style>@import`, no http(s):// or protocol-relative URLs. Inline fills/strokes only. This keeps it same-origin so both the CI privacy guard (greps dist/ incl. *.svg) and CSP `img-src 'self'` pass unchanged. Keep it lightweight (well under ~1 KB).

In index.html `<head>` (after the viewport meta, before `<title>`), add the favicon link. Use Vite's base-aware token so it resolves under the project base path: `<link rel="icon" type="image/svg+xml" href="%BASE_URL%favicon.svg" />`. Vite substitutes `%BASE_URL%` with `/battery-calculator/` at build time. A hand-written literal href is NOT auto-prefixed with base, which is exactly the trap that caused the original deploy bug — so use `%BASE_URL%`.

Do NOT modify src/constants/csp.ts. `img-src 'self'` already covers a same-origin favicon.
