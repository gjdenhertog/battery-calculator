<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build → shipped dist/ | A dependency could inject a third-party URL at build time; the guard scans the built artifact |
| CI gate → Pages deploy | Deploy must not happen unless lint/test/guard all pass |
| browser → any origin | The live page must make zero outbound requests with user data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Information Disclosure | build-time-injected analytics/CDN/font URL in dist/ | mitigate | CI privacy guard greps dist/ for external https:// and fails the build (D-04); scans BUILT output not source |
| T-01-03-02 | Information Disclosure | error-reporting library (Sentry/Rollbar/Bugsnag) added later | mitigate | CI step greps package.json for sentry|rollbar|bugsnag and fails (PRIV-03) |
| T-01-03-03 | Tampering | deploy of a broken/failing build | mitigate | deploy job `needs: ci`; a failing ci job blocks deploy (ROADMAP criterion 4) |
| T-01-03-04 | Elevation of Privilege | over-broad workflow token permissions | mitigate | minimal permissions: contents read, pages write, id-token write only (Pitfall 4) |
| T-01-03-SC | Tampering | GitHub Actions third-party deploy action supply chain | mitigate | official actions/* chain only at locked versions (CLAUDE.md); no peaceiris/JamesIves third-party action |
</threat_model>
