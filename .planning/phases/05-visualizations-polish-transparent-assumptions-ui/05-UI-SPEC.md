# Phase 5 — UI Design Contract

> Visual and interaction contract for charts, the transparent-assumptions panel,
> Dutch-copy tooltips, mobile reflow, and the honest-terminology audit.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope note:** Phase 5 adds uPlot charts and a transparency panel inside the already-
> populated `#results-region`. It does NOT change the Phase 4 comparison table, the
> battery picker, the drop zone, or the saldering disclaimer. All tokens and color slots
> are inherited unchanged from Phase 1 tokens.css and Phase 4 UI-SPEC. uPlot (~22 KB gz)
> is the only new runtime dependency this phase introduces.

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | CLAUDE.md — vanilla TS + Vite, no component library |
| Preset | not applicable | — |
| Component library | none | CLAUDE.md constraint |
| Icon library | none — Unicode glyphs only | Phase 4 pattern; no new icons |
| Font | System-font stack (`--font-sans` token) | tokens.css (locked Phase 1) |

**Chart library:** uPlot `^1.6.32` — LOCKED (CLAUDE.md + 05-CONTEXT.md). Renders to `<canvas>`.
No alternative is in scope. uPlot renders via canvas; it requires no CSP relaxation because
it does not inject inline styles or fetch external assets.

**No shadcn.** No component registry. All styling via CSS classes + CSS custom properties.

### Per-Battery Color Slots in Charts (COMP-04)

All chart series colors come from `colorFor(batteryId, orderedSelection)` which returns
`var(--color-battery-N)` CSS custom property strings. These are resolved to their hex
values at chart-mount time via `getComputedStyle`. Never hard-coded hex in chart code.

| Slot | CSS Custom Property | Hex | uPlot series stroke |
|------|--------------------|----|---------------------|
| 1 | `--color-battery-1` | `#2563eb` | Blue — Sessy 5 kWh (default) |
| 2 | `--color-battery-2` | `#16a34a` | Green |
| 3 | `--color-battery-3` | `#d97706` | Amber |
| 4 | `--color-battery-4` | `#9333ea` | Purple |
| 5 | `--color-battery-5` | `#e11d48` | Rose |

**Monthly bars chart:** Each battery group uses its battery color slot at 80% opacity
(`rgba` or CSS `color-mix(in srgb, var(--color-battery-N) 80%, transparent)`) for full
months. Partial months ("deels") use 40% opacity of the same battery color — no new
color token; same variable, lower alpha. This opacity approach is the chosen treatment
for D-05 (see Interaction Contracts below).

**Flow chart series:** Four series (import, export, charge, discharge) for ONE battery.
They use the battery's assigned color slot plus three neutral tones:

| Series | Color | Rationale |
|--------|-------|-----------|
| Grid import (`residualImportKwh`) | `--color-battery-N` (battery's slot color) | Primary metric — stands out |
| Grid export / teruglevering (`residualExportKwh`) | `--color-text-muted` (#71717a) | Secondary — energy leaving the house |
| Battery charge (`chargedKwh`) | `#16a34a` (green, hardcoded as fallback series color) | Energy stored — green = positive |
| Battery discharge (`dischargedKwh`) | `#d97706` (amber, hardcoded as fallback) | Energy released |

**CSP note:** uPlot color values are passed as JavaScript strings in the chart config — NOT
as inline CSS `style=` attributes. uPlot renders to canvas; canvas draw calls are not
subject to `style-src 'self'`. This is CSP-safe.

## Component Inventory

### 1. Monthly Self-Consumption Grouped-Bar Chart (VIZ-01)

**Container:** `<section class="chart-section" aria-label="Maandelijks zelfverbruik">` placed
inside `#results-region`, below the comparison table block and above the flow chart section.

**Section heading:** `<h2 class="results-section-heading">Zelfverbruik per maand</h2>`
at `--font-size-heading` (20px), semibold.

**Chart canvas wrapper:**
```html
<div class="chart-wrapper" id="chart-monthly">
  <!-- uPlot mounts its canvas here -->
</div>
```

CSS: `width: 100%; height: 280px; background-color: var(--color-bg)`.
On screens ≤ 480px: `height: 220px` (auto-shortened — see Mobile Reflow).

**uPlot configuration:**
- Chart type: grouped bars (uPlot 1.6+ bar plugin or native bar series)
- x-axis: months (e.g. "jan '25", "feb '25") derived from Europe/Amsterdam local months
  via `@date-fns/tz` TZDate bucketing (pure function, node-env testable)
- y-axis: kWh, labeled "kWh" at axis, ticks via `formatKwh()` adapted for uPlot
- One bar series per selected battery, colored via `colorFor()` at mount time
- Full months: battery color at 80% opacity
- Partial months: same battery color at 40% opacity + "(deels)" annotation drawn as
  a `<span class="chart-partial-label">` below the x-axis label (not inside the canvas)
- Legend: inline battery name + swatch below the canvas (rendered as HTML, not uPlot
  native legend, for CSP safety and DOM accessibility)
- No extrapolation: bar height = actual shifted kWh for the real (partial) days recorded

**Sparse-data note** (D-06): When `< 2` full months exist in the dataset, a
`<p class="chart-sparse-note">` appears below the chart canvas:
```
Weinig data — je hebt minder dan twee volledige maanden geüpload.
Alle staven tonen de werkelijke data, niet een prognose.
```
Styled: `--font-size-label` (14px), `--color-text-muted`, `--color-surface` background,
`--space-sm` padding, border-radius 4px.

**Pure data helper:** `src/domain/bucket-by-month.ts` — a pure function that takes
`SimResult.trace` (array of `TraceRow`) and returns `MonthBucket[]` with month label,
`shiftedKwh`, and `isPartial: boolean`. Testable in node env without uPlot or DOM.

### 2. Sample-Week Energy Flow Step-Line Chart (VIZ-02 / VIZ-03)

**Container:** `<section class="chart-section" aria-label="Energiestroom voorbeeldweek">`
placed inside `#results-region`, below the monthly bars chart.

**Section heading + battery dropdown:**
```html
<div class="chart-section__header">
  <h2 class="results-section-heading">Energiestroom voorbeeldweek</h2>
  <label class="chart-battery-select__label" for="flow-chart-battery">
    Batterij:
    <select id="flow-chart-battery" class="chart-battery-select">
      <!-- one <option> per selected battery; swatch color via data-slot -->
    </select>
  </label>
</div>
```

The `<select>` defaults to the first selected battery (Sessy 5 kWh, D-01). Options list
battery names only (no color swatches inside `<select>` — native select cannot render
custom DOM). Adjacent to the heading: minimal, functional.

**Week caption:**
```html
<p class="chart-week-caption">
  Voorbeeldweek: {DD maand}–{DD maand yyyy} — de week met de meeste teruglevering in je data.
</p>
```
Example: "Voorbeeldweek: 8–14 juni 2025 — de week met de meeste teruglevering in je data."
Styled: `--font-size-label` (14px), `--color-text-muted`. Updated reactively on battery
dropdown change if different batteries have different highest-teruglevering weeks.

**Chart canvas wrapper:**
```html
<div class="chart-wrapper" id="chart-flow">
  <!-- uPlot mounts its canvas here -->
</div>
```

CSS: `width: 100%; height: 320px`.
On screens ≤ 480px: `height: 260px; overflow-x: auto` (chart scrolls horizontally within
its own wrapper — headline numbers outside the wrapper are unaffected, D-13).

**uPlot configuration:**
- Chart type: 4 step-line series (VIZ-03 — stepped paths, `drawStyle: 1` or equivalent
  uPlot stepped-line setting; NEVER `drawStyle: 0` smooth interpolation)
- x-axis: time (UTC timestamps from `TraceRow.timestamp`), displayed in Europe/Amsterdam
  local time using uPlot's `tzDate` option with the IANA zone "Europe/Amsterdam"
- y-axis: kWh, ticks via `formatKwh()` adapter
- Series (in legend order):

  | # | Series name | Data field | Color |
  |---|-------------|-----------|-------|
  | 1 | Grid import | `residualImportKwh` | Battery's slot color (primary) |
  | 2 | Teruglevering | `residualExportKwh` | `--color-text-muted` (#71717a) |
  | 3 | Laden | `chargedKwh` | Green (#16a34a) |
  | 4 | Ontladen | `dischargedKwh` | Amber (#d97706) |

- Week selection heuristic: `src/domain/select-representative-week.ts` — pure function
  returning the 7-day window (Mon–Sun, Europe/Amsterdam) with the highest sum of
  `residualExportKwh`. Testable in node env. Returns `{ startTs: number, endTs: number, weekLabel: string }`.
- Legend: inline HTML below canvas (battery swatch + series name per row), NOT uPlot
  native legend (DOM-rendered for CSP safety)

**Interaction:** When user changes the `<select>` dropdown, the chart re-renders with:
1. New `trace` data from the corresponding `SimResult` for the selected battery
2. Series color 1 updated to the new battery's slot color
3. Week caption updated (if the representative week differs)
4. Battery dropdown options color-coded via `data-slot="{N}"` attribute (the CSS
   `.chart-battery-select option[data-slot="N"]` cannot set text-color in `<option>` in
   most browsers — the slot data is for future enhancement; the `<select>` label shows
   the battery name only)

### 4. Technical-Term Tooltips (UX-03 / D-11)

**Trigger pattern:** Any tooltipped term is wrapped in a `<abbr>` or `<span>` with class
`.term-tooltip`:
```html
<span class="term-tooltip" data-tooltip="Rendement van het laden én ontladen samen,
uitgedrukt als percentage van de ingeslagen energie die je terugkrijgt.">
  round-trip rendement
</span>
```

**CSS (in `src/styles/tooltips.css`):**
```css
.term-tooltip {
  text-decoration: underline;
  text-decoration-style: dotted;
  text-underline-offset: 3px;
  cursor: help;
  position: relative;
}

.term-tooltip::after {
  content: attr(data-tooltip);
  position: absolute;
  bottom: calc(100% + var(--space-xs));
  left: 0;
  background-color: var(--color-text);
  color: var(--color-bg);
  font-size: var(--font-size-label);
  font-weight: var(--font-weight-regular);
  line-height: 1.4;
  padding: var(--space-xs) var(--space-sm);
  border-radius: 4px;
  white-space: normal;
  max-width: 280px;
  min-width: 160px;
  z-index: 10;
  pointer-events: none;
  opacity: 0;
  transition: none; /* no CSS transitions — style-src 'self' safe */
}

/* Desktop hover */
.term-tooltip:hover::after {
  opacity: 1;
}

/* Keyboard focus */
.term-tooltip:focus-visible::after {
  opacity: 1;
}

/* Mobile tap-toggle via JS class */
.term-tooltip--open::after {
  opacity: 1;
}
```

**Interaction (mobile / touch):** A `touchstart` event listener on `document` toggles
class `.term-tooltip--open` on the tapped `.term-tooltip` element and removes it from
all other `.term-tooltip` elements. A `touchstart` anywhere else dismisses open tooltips.
This is the tap-toggle mechanism (D-11) — implemented in JS, no inline event handlers.

**Keyboard:** `tabindex="0"` on each `.term-tooltip` span. `:focus-visible::after` shows
the tooltip. `Escape` keypress removes `.term-tooltip--open` and blurs the element.

**Tooltipped terms (full list — D-11):**

| Term displayed | `data-tooltip` content |
|----------------|------------------------|
| zelfverbruik | De energie die je thuis opwekt en direct zelf gebruikt, in plaats van aan het net te leveren. |
| teruglevering | Stroom die jouw huis terugstuurt naar het net — bij jou zichtbaar als export op de P1-meter. Wij meten dit; we meten geen zonneopwekking rechtstreeks. |
| marginale benutting | Hoeveel kWh de batterij per kWh capaciteit heeft verschoven. Een hoge waarde betekent dat de batterij goed wordt benut; een lage waarde dat er weinig teruglevering was om op te vangen. |
| round-trip rendement | Het percentage van de ingeslagen energie dat je terugkrijgt bij ontlading. Een rendement van 85% betekent dat 15% verloren gaat aan warmte bij het laden én ontladen samen (√0,85 ≈ 92% per richting). |
| diepteontlading (DoD) | Het maximale percentage van de batterijcapaciteit dat gebruikt mag worden. Een batterij van 5 kWh met 90% DoD slaat nooit meer dan 4,5 kWh op. |
| saldering | De Nederlandse regeling waarmee je geëxporteerde stroom mag verrekenen met geïmporteerde stroom op je jaarrekening. Saldering wordt per 2027 volledig afgeschaft. |
| verschoven kWh | De energie die de batterij heeft verplaatst van een moment van teruglevering naar een moment van import — de kern van wat een thuisbatterij doet. |

**XSS safety:** `data-tooltip` content is always author-defined (hard-coded in source), never
interpolated from user-uploaded CSV data. Rendered via CSS `content: attr(data-tooltip)` —
no `innerHTML` used for tooltip display.

with `--space-md` (16px) horizontal padding — established Phase 1.

**Battery picker grid:** Already collapses to 1 column below 480px (Phase 4 `battery-picker.css`).

**Transparency panel:** Full-width block, `<details>` layout is inherently single-column.

## Interaction Contracts

### Monthly Bar Chart Interactions

| Interaction | Behavior |
|-------------|----------|
| New `SimResult[]` arrives (worker completes) | Re-run `bucketByMonth()` on all battery traces; call `uPlot.setData()` with new bar series; update partial-month annotations |
| Selected batteries change | Rebuild chart series array in `colorFor()` order; re-render legend below canvas |
| Period changes | New `SimResult[]` → same as above (charts react to state, not to period directly) |
| Viewport resize | uPlot `setSize({ width: wrapper.offsetWidth, height: ... })` called in a `ResizeObserver` on the chart wrapper; debounced 100ms |

### Flow Chart Interactions

## CSP Compatibility

All Phase 5 components inherit the Phase 1 / Phase 4 CSP contract. Phase 5 checklist:

| CSP directive | Value | Phase 5 compliance |
|--------------|-------|--------------------|
| `style-src` | `'self'` | All styles in `.css` files; zero inline `style=` attributes; uPlot color strings in canvas JS config (canvas draw calls are not subject to `style-src`) |
| `script-src` | `'self'` | uPlot is bundled by Vite; no CDN script tag; no `eval()` |
| `worker-src` | `blob:` | Unchanged from Phase 4 (Comlink workers) |
| `connect-src` | `'none'` | uPlot renders to canvas — no fetch; no external data; all chart data comes from in-memory `SimResult.trace` |
| `font-src` | `'self'` | uPlot font config uses the resolved system font stack — no web font fetch |
| `img-src` | `'self'` | No images; chart legend swatches are CSS `background-color` on `<span>` elements, not `<img>` |

**uPlot tooltip note:** uPlot's built-in tooltip plugin would inject an inline `style=` on
the tooltip element (absolute positioning). Do NOT use uPlot's built-in tooltip.
Use CSS `::after` pseudo-element tooltips instead (already specified above for term
tooltips). Chart value readouts on hover (if needed) must be rendered as a DOM element
with a CSS class, not via `element.style.left = ...` in JS.

**uPlot resize note:** uPlot's `setSize()` call sets canvas width/height attributes
(HTML attributes, not style attributes) — CSP-safe.
