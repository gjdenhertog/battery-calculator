<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| parsed CSV samples → signal store | user-derived numeric data enters reactive state |
| main thread → Comlink worker | samples + battery configs cross the worker boundary |
| custom battery form values → activeBatteries | untrusted numeric input crosses into runComparison |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Denial of Service | crafted huge custom-battery numbers reaching runComparison | mitigate | activeBatteries only appends a custom battery with nominalCapacityKwh > 0; numeric bounds enforced at the form layer (plan 04-04) via input min/max before the value reaches the store |
| T-04-07 | Tampering | out-of-order worker results overwriting newer state | mitigate | generation counter in scheduleRecompute discards superseded results (RESEARCH Open Q3) — only the newest myGen writes simResults |
| T-04-08 | Information Disclosure | worker fetch/exfiltration of CSV samples | mitigate | `connect-src 'none'` (plan 04-01) applies inside the worker; Comlink uses postMessage not fetch; samples never leave the browser |
| T-04-09 | Tampering | effect/subscription leak from un-disposed effects | mitigate | app-state itself creates NO effects (only signals/computeds + a debounced async fn); UI plans own effect disposal (RESEARCH Pitfall 3) |
</threat_model>
