    - That N matches the slot the comparison table assigns to the custom row for the same active selection.
    - When no valid custom battery is active (customBattery null / capacity invalid), the custom card's swatch is hidden (no visible color dot).
  </behavior>
  <action>
    1. In `buildCustomCard()`: add a `.battery-card__swatch` span (mirroring the catalog swatch element from `buildSpecCard`). Append it so it reads as the custom card's color dot — place it adjacent to the expand button (e.g. inside a small header row, or before `expandBtn`), consistent with how catalog cards show swatch next to the name. Give it an initial `hidden = true` and a class marker so the effect can find it (e.g. add a stable class `custom-card__swatch` in addition to `.battery-card__swatch`, OR query it via the custom card scope). Do NOT set any color via `style` — color comes only from `battery-swatch--N` classes (style-src 'self' CSP). Return the swatch element (or expose the custom `<li>`) so the main init function's effect can update it.

    2. In `initBatteryPicker()`, locate the custom card after `buildCustomCard()` and query its swatch (`customCard.querySelector('.battery-card__swatch')`). Extend the EXISTING reactive `effect(...)` (the one tracking `selectedBatteries`) — or, cleaner, push an ADDITIONAL effect into `_disposeFns` — that reads `customBattery.value` and `activeBatteries.value`:
       - If `customBattery.value` is non-null AND `(customBattery.value.nominalCapacityKwh ?? 0) > 0`: compute `slot = colorSlotFor('custom', activeBatteries.value.map(b => b.id))`, remove `battery-swatch--1..5`, add `battery-swatch--${slot}`, and set the swatch `hidden = false`.
       - Else: set the swatch `hidden = true` (and optionally strip the slot classes).
       Reading `activeBatteries.value` (a computed over selectedBatteries + customBattery) ensures the effect re-runs when EITHER selection order or the custom battery changes, keeping the picker swatch in lockstep with the table.

    3. Effect disposal discipline (Pitfall 3): if you add a new effect, push its dispose fn into the existing `_disposeFns` array so `teardownBatteryPicker()` cleans it up. Do NOT create the effect inside another effect or inside an event handler.

    4. XSS safety: the swatch carries no user text — it is class-only. No `.innerHTML`, no inline style.
  </action>
