<objective>
Build the battery spec-card picker (D-05) inside `#drop-zone-region`: render all 7 catalog
batteries as selectable cards with visible specs + color swatch, pre-check Sessy 5 (BATT-03),
cap selection at 5 with disabled cards + a Dutch note (BATT-05), and add an inline-expandable
custom battery card with 5 validated number fields (BATT-04, D-06). Discrete changes call
`scheduleRecompute(true)`; custom-field typing debounces (400ms, D-07).

Purpose: This is where the user chooses what to compare. It writes `selectedBatteries` and
`customBattery` signals; the color swatch uses `colorSlotFor` so identity color is consistent
with the table (COMP-04). XSS discipline (`.textContent`) and the no-inline-style CSP rule
(`.battery-swatch--N` classes) are load-bearing here because the custom battery name is
user-derived. Renders on mount even before any file is uploaded (UI-SPEC §8 — user explores
specs first).
Output: battery-picker module + CSS + jsdom tests.
</objective>
