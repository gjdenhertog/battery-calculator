<objective>
Close the MAJOR shared-container clobber gap (UAT Test 5 + Test 9) and the CONTENT saldering-copy gap (UAT Test 7).

Root cause (confirmed live): `src/main.ts` mounts BOTH `initPeriodControl(resultsRegion)` and `initComparisonTable(resultsRegion)` into the same `#results-region`. The comparison-table renderers (`renderEmpty`/`renderError`/`renderTable`) each call `container.innerHTML = ''` (comparison-table.ts lines 48, 56, 339). The table effect runs immediately on load (empty state) and on every recompute, destroying the entire period-control `<section>` (Analyseperiode heading, Van/Tot inputs, coverage indicator, framing note). This escaped CI because each jsdom test mounts its component in an isolated container — the layering bug only manifests in the integrated live DOM.

Secondary: `SALDERING_DISCLAIMER_COPY` is factually stale (claims a "64% afbouw vanaf 2026" and a "50% saldering floor t/m 2030" — both describe a rejected proposal that never took effect).

Purpose: Make the period control survive table renders so period narrowing (Test 9) becomes exercisable, and ship factually correct saldering copy.
Output: A dedicated table mount node, scoped innerHTML clearing, a regression integration test, corrected disclaimer copy, and updated copy-substring assertions.
</objective>
