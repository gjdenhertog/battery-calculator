<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → DOM (custom battery NAME field) | NEW untrusted-text surface introduced by D-02. The free-text name flows into the card label and (via customBatteries) into the comparison-table column header and swatch legend. Must be inserted as `.textContent` / set as `input.value`, never via `innerHTML`, and must not produce inline styles. |
| user → simulation input (custom numeric fields) | Numeric fields parsed via parseFloat with range guards; only nominalCapacityKwh > 0 enters activeBatteries (carried from 06-01 filter). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02 | Tampering/XSS (Injection) | custom name field → card label | mitigate | Render the name strictly via `.textContent` / `input.value`; never assign it to `.innerHTML`. The `Eigen batterij ${n}` default uses only a static prefix + integer. Test asserts a `<script>`-laden name yields zero `<script>` elements in the picker DOM (mirror existing T-04-13 XSS test). |
| T-06-03 | Tampering | custom numeric inputs → sim input | mitigate | parseFloat + min/max range checks; only nominalCapacityKwh > 0 reaches activeBatteries (06-01 guard). Empty/invalid drafts excluded from the cap and the compute. |
| T-06-04 | Tampering (CSP) | swatch color application | mitigate | Colors applied only via `.battery-swatch--N` CSS classes (D-10); test/grep asserts no `.style.` assignment in battery-picker.ts (`style-src 'self'`). |
| T-06-SC | Tampering | npm/pip/cargo installs | n/a | No new packages installed; no install tasks in this plan. |
