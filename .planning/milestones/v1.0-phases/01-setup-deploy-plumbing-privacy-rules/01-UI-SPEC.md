# Phase 1 — UI Design Contract

> Visual and interaction contract for the structural shell, design-token baseline, and privacy copy.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope note:** Phase 1 ships NO feature code. It establishes the 3-region structural shell
> (header / drop-zone region / results placeholder), the full CSS custom-property token system
> that Phases 2–5 inherit, the system-font stack, the mobile-first container, and the
> verbatim Dutch privacy promise. No "coming soon" copy anywhere (D-02).

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | CLAUDE.md — vanilla TS + Vite, no framework, no component library |
| Preset | not applicable | — |
| Component library | none | CLAUDE.md constraint |
| Icon library | none (Phase 1 has no icons) | default — no icons needed in structural shell |
| Font | System-font stack (see Typography) | D-06: no web fonts, satisfies `font-src 'self'` + `connect-src 'none'` CSP |

**Implementation:** Design tokens ship as a `:root` CSS custom-property block in
`src/styles/tokens.css`, imported by `src/styles/global.css`. No inline `style=`
attributes — required by the `no 'unsafe-inline'` CSP (D-03).

## Spacing Scale

Declared values — multiples of 4 only. Exposed as CSS custom properties on `:root`.

| Token | CSS Custom Property | Value | Usage |
|-------|--------------------|----|-------|
| xs | `--space-xs` | 4px | Icon gaps, tight inline padding |
| sm | `--space-sm` | 8px | Compact element spacing, label-to-input |
| md | `--space-md` | 16px | Default element spacing, card inner padding |
| lg | `--space-lg` | 24px | Section padding, between form rows |
| xl | `--space-xl` | 32px | Layout column gaps |
| 2xl | `--space-2xl` | 48px | Major section breaks (between shell regions) |
| 3xl | `--space-3xl` | 64px | Page-level top/bottom padding |

Exceptions:
- Touch target minimum: 44px height on any interactive element (WCAG 2.5.5). Applied
  as a one-off rule, not a token, because it is a constraint not a layout value.
- Drop-zone region minimum height: 120px. Defined as a local rule in the drop-zone
  component, not a spacing token, because it is a structural sizing constraint.

## Typography

System-font stack declaration (single CSS custom property):

```
--font-sans: ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont,
             "Segoe UI", Roboto, "Helvetica Neue", Arial, "Noto Sans",
             sans-serif, "Apple Color Emoji", "Segoe UI Emoji";
```

No web fonts. No `@font-face`. No external font fetch. CSP `connect-src 'none'` is
satisfied from day one (D-06).

| Role | CSS Custom Property | Size | Weight | Line Height | Usage |
|------|--------------------|----|--------|-------------|-------|
| Body | `--text-body` | 16px | 400 (regular) | 1.5 | Default prose, privacy promise |
| Label | `--text-label` | 14px | 400 (regular) | 1.4 | Secondary metadata, tagline |
| Heading | `--text-heading` | 20px | 600 (semibold) | 1.2 | App title in header (`<h1>`) |
| Display | — | — | — | — | Not used in Phase 1 |

**Weights declared: 2** — 400 (regular) and 600 (semibold). No other weights.
**Sizes declared: 3** — 14px, 16px, 20px. Display size is deferred to Phase 5 polish.

Font-size and weight values are exposed as CSS custom properties:

```css
--font-size-label:   14px;
--font-size-body:    16px;
--font-size-heading: 20px;
--font-weight-regular:  400;
--font-weight-semibold: 600;
```

## Color

Neutral-first palette — the app is a utility calculator, not a marketing surface.
All values exposed as CSS custom properties on `:root`. No inline color attributes.

| Role | CSS Custom Property | Value | Usage |
|------|--------------------|----|-------|
| Dominant (60%) | `--color-bg` | `#ffffff` | Page background, primary surface |
| Secondary (30%) | `--color-surface` | `#f4f4f5` | Drop-zone region background, results placeholder background, card surfaces |
| Border | `--color-border` | `#e4e4e7` | Region outlines, dividers |
| Text primary | `--color-text` | `#18181b` | All body copy, headings |
| Text muted | `--color-text-muted` | `#71717a` | Tagline, secondary labels, privacy promise |
| Accent (10%) | `--color-accent` | `#2563eb` | Reserved: primary CTA button (Phase 2+), active/focused interactive state rings |
| Destructive | `--color-destructive` | `#dc2626` | Reserved: parse error messages (Phase 2), destructive confirmations only |

Accent (`#2563eb`) is reserved for: primary CTA button label and border, and keyboard
focus rings on interactive elements. It is NOT used for decorative purposes, icons, or
section backgrounds.

Destructive (`#dc2626`) is reserved for: error states exposed by PRIV-03 (parse errors
shown to the user but never sent off-device). Not used in Phase 1 UI (no parsing yet),
but the token is declared now so Phase 2 can use it without touching tokens.css.

**Dark mode:** Not in scope for v1. No `prefers-color-scheme` media query in Phase 1.

### Per-Battery Color Slots

Phase 4 requires a consistent color per selected battery across table and charts (COMP-04).
The slots are declared now as CSS custom properties so Phase 4's `colorFor(batteryId)` helper
reads from the token system rather than hard-coding values.

Up to 5 batteries can be compared simultaneously (BATT-05).

| Slot | CSS Custom Property | Value | Notes |
|------|--------------------|----|-------|
| Battery 1 | `--color-battery-1` | `#2563eb` | Blue — same hue as accent; distinct at 600 |
| Battery 2 | `--color-battery-2` | `#16a34a` | Green |
| Battery 3 | `--color-battery-3` | `#d97706` | Amber |
| Battery 4 | `--color-battery-4` | `#9333ea` | Purple |
| Battery 5 | `--color-battery-5` | `#e11d48` | Rose |

All 5 slots are perceptually distinct at 100% opacity, remain distinguishable on the
`--color-surface` background, and pass WCAG AA contrast against `--color-bg` at the
sizes used in charts (minimum 3:1 for non-text graphical elements).

## Shell Structure Contract

Three regions, rendered in DOM order, styled as a vertical single-column layout.
This is the mount-point contract Phases 2–5 fill in place (D-01).

### Region 1 — Header

- Element: `<header role="banner">`
- Contains: `<h1>` with app title + `<p>` with one-line Dutch tagline
- App title: **"Thuisbatterij Calculator"** (literal, locked)
- Tagline: at planner/executor discretion — honest, non-financial, Dutch. Example:
  "Bereken hoeveel energie een thuisbatterij jouw huis zou besparen."
  (Claude's Discretion per CONTEXT.md — planner chooses exact wording)
- Background: `--color-bg`
- Bottom border: 1px `--color-border`
- Padding: `--space-md` vertical, `--space-lg` horizontal

### Region 2 — Drop-Zone Region

- Element: `<main id="drop-zone-region">`
- Phase 1 content: privacy promise only (D-02 — no "coming soon" copy)
- Background: `--color-surface`
- Border: 1px dashed `--color-border`, border-radius 8px
- Min height: 120px
- Padding: `--space-lg`

**Privacy promise — verbatim, locked (PRIV-02 / ROADMAP Phase 1 criterion 5):**

```
Je data blijft op je eigen apparaat — open je netwerktabblad en je ziet 0 verzoeken na het laden
```

Rendering rules:
- Rendered as a `<p>` element inside Region 2
- Font: `--font-size-body` (16px), weight `--font-weight-regular` (400)
- Color: `--color-text-muted`
- Must be visible in the drop-zone region, not in a footer or aside
- Must survive as a DOM node through Phase 2's real drop-zone replacement —
  Phase 2 keeps this `<p>` and adds the file-input controls around it

### Region 3 — Results Placeholder

- Element: `<section id="results-region" aria-label="Vergelijkingsresultaten">`
- Phase 1 content: empty — no copy, no placeholder text (D-02)
- Background: `--color-surface`
- Border: 1px solid `--color-border`, border-radius 8px
- Min height: 80px
- Padding: `--space-lg`
- Visibility: rendered in DOM but visually bare (styled placeholder, not hidden)

### Container

- Element: `<div class="container">` wrapping all three regions
- Max width: 720px
- Margin: `0 auto`
- Padding: `--space-lg` horizontal (collapses to `--space-md` below 480px)
- Viewport meta: `<meta name="viewport" content="width=device-width, initial-scale=1">`
- Box-sizing reset: `*, *::before, *::after { box-sizing: border-box }` in `global.css`

## Mobile-First Baseline (D-07)

Required in Phase 1 to avoid Phase 5 layout retrofit.

| Breakpoint | Width | Behavior |
|------------|-------|----------|
| Default (mobile) | 0px+ | Single column, `--space-md` horizontal container padding |
| Tablet+ | 480px+ | `--space-lg` horizontal container padding |
| Content cap | 720px | Container max-width; centered on wider screens |

No horizontal scroll at 375px viewport width. Verified by manual browser check at 375px.
No media-query-dependent font changes in Phase 1 — the 3-size type scale works at all widths.

## CSP Compatibility Contract

Every visual decision in this spec must be compatible with the maximal-lockdown CSP (D-03):

| CSP directive | Value | Design implication |
|--------------|-------|-------------------|
| `style-src` | `'self'` | No `style=` inline attributes anywhere; all styles in `.css` files |
| `font-src` | `'self'` | No Google Fonts, no CDN fonts; system-font stack only |
| `connect-src` | `'none'` | No external asset fetches; no font loading from external URLs |
| `img-src` | `'self'` | No external images; no `data:` URIs for decorative images |
| `script-src` | `'self'` | No inline `<script>` blocks; all JS in bundled module files |

**Implication for CSS custom properties:** All token values are plain CSS values (hex
colors, px values, unitless ratios). No `url()` references to external assets.
No `@import url(...)` in any stylesheet.
